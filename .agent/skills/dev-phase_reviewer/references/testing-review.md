# 测试阶段 Review 清单

适用阶段: `testing`

## 覆盖率检查

```bash
# Python
uv run pytest --cov=app --cov-report=term-missing

# TypeScript
npm run test:coverage
```

- [ ] **总覆盖率 ≥ 80%** — 最低要求
- [ ] **核心业务逻辑覆盖率 ≥ 90%** — Service 层、关键算法
- [ ] **无未覆盖的关键路径** — 主要业务流程都有测试

## 测试完整性

### 单元测试

- [ ] **Service 层** — 所有 public 方法有测试
- [ ] **工具函数** — 所有 utility 函数有测试
- [ ] **模型验证** — Pydantic/Zod schema 验证有测试
- [ ] **边界情况** — 空值、空列表、极端值有测试
- [ ] **错误路径** — 异常/错误场景有测试

### 集成测试

- [ ] **API 端点** — 所有 CRUD 端点有测试
- [ ] **数据库操作** — 创建/读取/更新/删除有测试
- [ ] **认证流程** — 登录/注册/权限有测试

### E2E 测试（Playwright）

- [ ] **核心用户流程** — 主要业务流程有 E2E 测试
- [ ] **认证流程** — 登录/注册/登出有 E2E 测试
- [ ] **关键交互** — 表单提交、搜索、导航有 E2E 测试

## 测试质量

- [ ] **命名描述性** — 测试名称描述行为和预期结果，无 `test("works")`
- [ ] **AAA 模式** — Arrange → Act → Assert 结构清晰
- [ ] **测试隔离** — 测试之间无共享状态，无执行顺序依赖
- [ ] **Mock 正确** — 外部依赖（API、数据库）正确 Mock
- [ ] **无固定 sleep** — 使用 waitFor/retry 而非 sleep
- [ ] **无 console.log** — 测试代码中无调试语句
- [ ] **无 .only/.skip** — 无被跳过或独占的测试

## 测试数据

- [ ] **测试数据隔离** — 使用独立的测试数据库/fixture
- [ ] **无硬编码外部依赖** — 不依赖真实的第三方 API
- [ ] **数据清理** — 测试后清理创建的数据

## 审批标准

- ✅ **批准**: 覆盖率 ≥ 80%，所有测试通过，无质量问题
- ⚠️ **条件通过**: 覆盖率 70-80%，有轻微质量问题
- ❌ **阻止**: 覆盖率 < 70%，有失败的测试，关键路径无测试

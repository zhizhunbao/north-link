>
> 依赖: [需求分析文档](requirements.md)

## 1. 功能需求

### 1.1 模块 1: 加拿大货源智能比价中心 (P0 — MVP)

**用户故事**:

- US-101: As a 货源商, I want to 看到加拿大各平台的商品最低价, so that 我能快速找到最便宜的货源
- US-102: As a 货源商, I want to 按分类/排序筛选商品, so that 我能高效定位目标品类
- US-103: As a 货源商, I want to 收藏高利润商品, so that 我能快速回看和采购
- US-104: As a 货源商, I want to 手动录入线下/私人价格, so that 非公开渠道的价格也能纳入比价
- US-105: As a 货源商, I want to CSV 批量导入价格数据, so that 我能快速录入大量商品信息

**验收标准**:

- [ ] MVP 数据来源: 手动录入 + CSV 批量导入（V1.5 增加平台自动爬取: Amazon, BestBuy, Costco, Walmart, Canada Computers, Newegg）
- [ ] 商品分类: 显卡、电脑硬件、苹果产品、保健品、母婴、美妆、海鲜、家电
- [ ] 排序方式: 价格升序、利润率、历史低价、捡漏等级
- [ ] 商品标记: 全新、二手、官翻、清仓、急售
- [ ] 支持手动价格录入
- [ ] 支持 CSV 批量导入价格数据
- [ ] 数据更新: MVP 手动触发；V1.5 自动更新 4-6 次/天

### 1.2 模块 2: 国内价格与商户匹配中心 (P0 — MVP)

**用户故事**:

- US-201: As a 货源商, I want to 看到国内对应商品的实时售价, so that 我能评估利润空间
- US-202: As a 货源商, I want to 管理下游商户联系人, so that 我能快速联系出货
- US-203: As a 货源商, I want to 系统自动匹配最高报价商户, so that 我能最大化利润

**验收标准**:

- [ ] 国内数据源: 淘宝、拼多多、闲鱼、1688、抖音
- [ ] 展示: 最低零售价、最高收购价、批发价
- [ ] 商户字段: 名称、联系方式、主营品类、报价、合作等级、历史订单
- [ ] 商户分类: 显卡/算力 | 数码电子 | 保健品/母婴 | 生鲜/海鲜
- [ ] 自动匹配当前商品最高报价商户

### 1.3 模块 3: 智能利润计算器 (P0 — MVP 核心)

**用户故事**:

- US-301: As a 货源商, I want to 一键计算纯利润, so that 我能快速判断是否值得做
- US-302: As a 货源商, I want to 自定义税率和运费参数, so that 计算结果更准确

**验收标准**:

- [ ] 公式: 纯利润 = 国内售价 - 加拿大成本 - 国际运费 - 关税 - 清关费 - 杂费
- [ ] 内置参数: 电子产品 16%, 保健品/美妆 8%, 母婴/日用 10%, 生鲜 10%
- [ ] 输出: 单件成本、总利润、利润率、风险等级（低/中/高）
- [ ] 颜色标记: 绿色=高利润, 黄色=一般, 红色=不推荐
- [ ] 参数可自定义编辑
- [ ] 计算响应 < 3 秒

### 1.4 模块 4: 跨境物流与货代管理 (P0 — MVP)

**用户故事**:

- US-401: As a 货源商, I want to 管理多个货代的信息和报价, so that 我能选择最合适的物流方案
- US-402: As a 货源商, I want to 一键下单发货, so that 我不用反复和货代沟通
- US-403: As a 货源商, I want to 跟踪物流状态, so that 我能及时了解货物位置

**验收标准**:

- [ ] 货代字段: 名称、单价、时效、是否包税、是否上门取件、评级、历史记录
- [ ] 一键发货: 填写商品信息 → 自动算运费税费 → 生成取件码
- [ ] 物流方案推荐: 最便宜 | 最快 | 最稳(包税) | 批量最优
- [ ] 物流轨迹: 加拿大 → 清关 → 国内 → 签收

### 1.5 模块 5: 订单与资金管理 (P1)

**用户故事**:

- US-501: As a 货源商, I want to 记录每笔采购和销售订单, so that 我能追踪每笔交易
- US-502: As a 货源商, I want to 查看利润报表, so that 我能了解整体经营状况

**验收标准**:

- [ ] 采购订单: 商品、数量、成本、渠道、采购时间、付款/到货状态
- [ ] 销售订单: 出货对象、报价、运费、税费、利润、发货/清关/签收/回款状态
- [ ] 报表: 今日/本周/本月营收、总利润、毛利率、成本占比、单品排行
- [ ] 支持导出 Excel/PDF

### 1.6 模块 6: 每日智能推荐 (P0 — MVP 核心)

**用户故事**:

- US-601: As a 货源商, I want to 每天收到高利润商品推荐, so that 我无需分析直接执行

**验收标准**:

- [ ] 每日自动生成 TOP5 最赚钱商品
- [ ] 包含: 商品名称、加拿大最低价、国内最高售价、单件纯利润、推荐商户、推荐物流
- [ ] 推荐逻辑基于利润率 + 风险等级 + 历史成交

### 1.7 模块 7: 系统设置 (P1)

**用户故事**:

- US-701: As a 货源商, I want to 自定义系统参数, so that 系统适配我的实际情况

**验收标准**:

- [ ] 可编辑: 税率、运费模板、商户分类、货代信息
- [ ] 账号权限管理
- [ ] 数据备份与导出

### 1.8 模块 8: 用户认证 (P0 — MVP)

**用户故事**:

- US-801: As a 货源商, I want to 登录系统, so that 只有我能看到我的数据
- US-802: As a 货源商, I want to 修改密码, so that 我能保护账号安全

**验收标准**:

- [ ] 用户名 + 密码登录
- [ ] JWT Token 认证（access 30min + refresh 7d）
- [ ] 初始管理员通过 CLI seed 脚本创建
- [ ] 密码使用 bcrypt 哈希存储
- [ ] 登录失败后提示错误信息

## 2. 非功能需求

| 类别     | 要求                                               | 优先级 |
| -------- | -------------------------------------------------- | ------ |
| 易用性   | 全中文界面，一键操作，零技术门槛                   | P0     |
| 性能     | 页面加载 < 2 秒，利润计算 < 3 秒                   | P0     |
| 数据更新 | MVP: 手动录入 + CSV 导入；V1.5: 自动更新 4-6 次/天 | P0     |
| 安全性   | 商户信息、订单数据加密存储                         | P1     |
| 兼容性   | 电脑端 + 手机浏览器 (响应式)                       | P0     |

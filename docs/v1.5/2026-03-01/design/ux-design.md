## 1. 设计概述

V1.5 的核心 UX 改变：用户不再通过监控面板管理爬虫，而是通过 **AI 对话** 按需查询商品数据。Chat 是全系统最重要的交互界面。

### 1.1 设计原则

| 原则           | 说明                                                     |
| -------------- | -------------------------------------------------------- |
| **对话优先**   | Chat 是主入口，所有采集操作通过对话触发                  |
| **结果即操作** | 查询结果直接可操作（收藏、订阅、比价、打开链接）         |
| **渐进披露**   | 简单对话即可使用，高级功能（订阅管理、成本监控）按需展开 |
| **成本可见**   | 让用户感知采集成本（今日已用 X 次 / 100 次上限）         |
| **暗色延续**   | 延续 MVP 暗色主题，Chat 区域增加微妙的区分               |

### 1.2 改动范围

| 类型    | 页面                      | 改动内容                               |
| ------- | ------------------------- | -------------------------------------- |

#### 交互设计

- **发送消息**: Enter 发送，Shift+Enter 换行
- **流式输出**: AI 回复逐字显示（本地 LLM 通过 SSE 流式返回）
- **采集进度**: 实时更新各平台采集状态（SSE 推送）
- **结果操作**: 每个结果卡片底部有操作按钮行
- **上下文追问**: 自动携带上下文（"换个平台看看"、"只看 2000 以下的"）
- **快捷输入**: 输入框下方可选快捷建议（"比价"、"查利润"、"搜小红书"）
- **空状态**: 新对话显示欢迎语 + 功能说明 + 示例问题可点击
- **错误处理**: 采集失败时在对话中友好提示（"Amazon.ca 暂时无法访问，建议稍后重试"）
- **加载状态**: 骨架屏 + 进度条组合

## 4. 用户流程

### 4.1 首次使用 AI 助手

```
用户登录 → Dashboard
  → 看到 Chat 快捷入口卡片
  → 点击 [打开 →] 或 Sidebar「AI 助手」
  → 进入 Chat 页面
  → 看到欢迎语 + 功能示例
  → 点击示例 "帮我查 RTX 4090 在加拿大的价格"
  → AI 自动执行查询
  → 返回比价表格
  → 用户操作: 收藏 / 订阅 / 追问
```

### 4.2 跨境比价全流程

```
用户输入: "RTX 4090 在加拿大几个平台多少钱"
  → AI 识别意图: price_compare
  → 显示采集进度条 (4 个平台)
  → 平台逐个返回结果 (SSE 实时更新)
  → 全部完成 → 展示比价表格
  → 用户追问: "1688 批发价多少，利润多少？"
  → AI 识别意图: profit_calc
  → 采集 1688 → 计算利润 → 展示利润卡片
  → 用户: "帮我盯着 Amazon 这个，降价通知我"
  → AI 创建订阅 → 确认消息
```

### 4.3 社交平台搜索

```
用户输入: "小红书上最近大家都在推荐什么加拿大保健品"
  → AI 识别意图: social_search
  → 调用 MediaCrawler 搜索小红书
  → 显示采集进度
  → 返回 5-10 条笔记摘要卡片
  → 用户点击 [查看原文] → 打开小红书链接
```

### 4.4 订阅价格追踪

```
在对话中创建了订阅
  → 系统每日定时检查 (Celery Beat)
  → 价格变动超阈值
  → 创建通知
  → NotificationBell 红点 +1
  → 用户点击铃铛 → 展开 Drawer
  → 查看通知: "RTX 4090 降价 12% → $1,889"
  → 点击通知 → 跳转到对应比价详情
```

### 4.5 AI 不可用降级

```
用户进入 Chat 页面
  → 系统检测 Ollama 连接失败
  → 显示 Fallback UI
  → 用户手动输入关键词 + 选择平台
  → 点击搜索
  → 直接调用爬虫 → 返回结果表格
```

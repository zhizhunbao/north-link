# North Link V1.5 — AI 驱动按需数据采集 需求文档

> 2026-03-01 更新：核心交互从"全量定时采集"改为"AI 对话驱动的按需精准采集"。
> AI 使用本地大模型（Ollama），LLM 调用零成本。
> 用户通过 Chat Box 描述需求 → AI 解析意图 → 系统按需抓取 → 结构化返回。

## 1. 项目概述

### 1.1 背景

North Link V1.0（MVP）已完成核心功能：比价中心、商户管理、订单管理、物流管理、利润计算。但数据来源依赖手动录入和 CSV 导入，无法满足实际业务需求。

V1.5 目标：**通过 AI Chat 界面实现按需数据采集**。用户不需要配置复杂的爬虫参数，只需在对话框中用自然语言描述需求，系统自动确定抓取目标、执行采集、返回结构化结果。**只抓用户需要的数据，不盲目全量采集，控制成本。**

### 1.2 核心理念

```
❌ 旧模式: 定时全量采集所有平台所有品类 → 成本高、数据冗余、抓不起
✅ 新模式: 用户对话 → AI 理解意图 → 按需精准采集 → 结构化返回 → 可选订阅追踪
```

### 1.3 目标

1. 提供 AI Chat 界面，用户用自然语言描述采集需求
2. AI 解析意图，确定目标平台、关键词、采集范围
3. 系统按需执行精准采集（仅抓用户要的数据）
4. 结果在对话中结构化展示（表格、卡片、图表）
5. 用户可"订阅"特定商品/关键词的价格追踪（才触发定时采集）
6. 价格变动通知（仅订阅商品，非全量）

### 1.4 范围

| 范围内                     | 范围外       |
| -------------------------- | ------------ |
| AI Chat 对话界面           | AI 智能定价  |
| 按需精准数据采集           | APP 端推送   |
| 用户订阅制价格追踪         | 多用户协作   |
| 价格变动告警（仅订阅商品） | 平台账号管理 |
| 多平台比价结果展示         | 自动下单     |
| 社交平台数据按需查询       | 全量定期采集 |

## 2. 目标用户

### 2.1 用户画像

- **主要用户**: 跨境电商从业者（代购/批发），需要实时比价来做采购决策
- **使用频率**: 每日多次通过对话查询价格
- **核心诉求**: 快速获取指定商品跨平台价格，发现价格洼地，跟踪感兴趣的商品

### 2.2 用户场景（对话驱动）

| 场景         | 对话示例                                                   | 系统行为                                                    |
| ------------ | ---------------------------------------------------------- | ----------------------------------------------------------- |
| 跨平台比价   | "帮我查一下 RTX 4090 在加拿大几个平台的价格"               | AI → 采集 Amazon.ca + BestBuy + Walmart + Costco → 对比表格 |
| 中加比价     | "这款显卡在 1688 上批发价多少？加拿大卖多少？利润有多少？" | AI → 采集 1688 批发价 + CA 零售价 → 计算利润                |
| 社交热度查询 | "小红书上最近大家都在推荐什么保健品？"                     | AI → MediaCrawler 搜小红书 → 返回热门笔记摘要               |
| 订阅追踪     | "帮我盯着这个商品，降价超过 10% 通知我"                    | AI → 创建订阅任务 → 定时采集此商品 → 达阈值通知             |
| 闲鱼淘货     | "帮我看看闲鱼上有没有便宜的 iPhone 15 Pro"                 | AI → ai-goofish 搜闲鱼 → AI 过滤 → 返回优质商品列表         |
| 趋势分析     | "抖音上最近什么品类的带货视频最火？"                       | AI → MediaCrawler 搜抖音 → 返回热度排行                     |

## 3. 功能需求

### 3.1 核心功能

#### F1: AI Chat 对话界面 ⭐ 核心入口

**优先级: P0 — 最高优先级**

- Dashboard 或独立页面提供 Chat Box
- 支持自然语言输入，中英文均可
- 对话历史保留（当前会话 + 历史会话列表）
- AI 回复支持富文本：表格、商品卡片、价格图表、链接
- 支持上下文追问（"换个平台再看看"、"这个能再便宜吗"）
- 显示采集进度（"正在从 Amazon.ca 搜索..."）
- 每条结果可操作：收藏、订阅追踪、加入比价

#### F2: AI 意图解析引擎

**优先级: P0**

AI 解析用户输入，输出结构化采集任务：

```json
{
  "intent": "price_compare",
  "keywords": ["RTX 4090"],
  "platforms": ["amazon_ca", "bestbuy_ca", "walmart_ca"],
  "filters": { "price_max": 2500, "currency": "CAD" },
  "action": "search_and_compare"
}
```

支持的意图类型：

| 意图             | 触发示例                       | 系统动作                |
| ---------------- | ------------------------------ | ----------------------- |
| `price_search`   | "查一下 XX 的价格"             | 指定平台搜索            |
| `price_compare`  | "比较一下 XX 在几个平台的价格" | 多平台采集 + 对比       |
| `profit_calc`    | "这个利润多少"                 | 采集两端价格 + 计算利润 |
| `social_search`  | "小红书上 XX 怎么样"           | 社交平台搜索            |
| `trend_analysis` | "最近什么火"                   | 社交平台热度分析        |
| `subscribe`      | "盯着这个商品"                 | 创建订阅追踪            |
| `xianyu_search`  | "闲鱼上有没有便宜的 XX"        | 闲鱼搜索 + AI 过滤      |

#### F3: 按需数据采集引擎

**优先级: P0**

各平台采集能力（仅在用户请求时触发）：

**加拿大电商平台：**

| 平台       | 技术方案                                                              | 反爬难度 |
| ---------- | --------------------------------------------------------------------- | -------- |
| Amazon.ca  | `amazon-scraper` + Playwright stealth；逆向搜索 API                   | ⭐⭐⭐   |
| BestBuy.ca | **官方 Developer API** — 无需爬虫                                     | ⭐       |
| Walmart.ca | `scrapfly-scrapers` (httpx + parsel)；逆向内嵌 JSON API               | ⭐⭐     |
| Costco.ca  | `costco-monitor` + Playwright stealth；**Akamai 反爬，商业 API 兜底** | ⭐⭐⭐⭐ |

**中国电商平台：**

| 平台   | 技术方案                                         | 反爬难度 |
| ------ | ------------------------------------------------ | -------- |
| 1688   | `Alibaba-CLI-Scraper` + Apify 1688 Actors        | ⭐⭐     |
| 淘宝   | `taobao_mcp` / `InfoSpider`；需登录态 + 滑块验证 | ⭐⭐⭐⭐ |
| 京东   | `PriceDive` (覆盖京东+淘宝+拼多多)               | ⭐⭐     |
| 拼多多 | `pinduoduo-scraper-example` / PriceDive          | ⭐⭐⭐   |

**社交平台：**

| 平台                                | 技术方案                               | 反爬难度 |
| ----------------------------------- | -------------------------------------- | -------- |
| 小红书 · 抖音 · 快手 · B站 · 微博等 | `NanmiCoder/MediaCrawler` (34k+ ⭐)    | ⭐⭐     |
| 闲鱼                                | `ai-goofish-monitor` (Playwright + AI) | ⭐⭐⭐   |

采集字段标准（电商）:

- `product_name`: 商品名称
- `sku`: 平台 SKU / ASIN / 商品 ID
- `price`: 当前售价
- `original_price`: 原价（如有折扣）
- `currency`: 货币代码 (CAD / CNY)
- `stock_status`: 库存状态
- `url`: 商品链接
- `image_url`: 商品主图
- `rating`: 评分
- `review_count`: 评论数
- `scraped_at`: 采集时间

#### F4: 订阅追踪（替代全量定时采集）

**优先级: P1**

**核心设计**：不做全量定时采集，只对用户明确订阅的商品/关键词做定时追踪。

- 用户在对话中说"帮我盯着" / "订阅" / "追踪" → 创建订阅
- 订阅维度：具体商品 URL / 关键词 + 平台
- 订阅频率：每日 1-2 次（节省成本）
- 价格变动超阈值时通知（默认 ±10%，可自定义）
- 订阅管理页面：查看/暂停/删除订阅
- 订阅数量限制：初期每用户最多 20 个（控制成本）

#### F5: 采集结果展示

**优先级: P1**

- 对话内嵌: 表格对比、商品卡片（图片+价格+链接）
- 历史查询: 可回看之前的对话和采集结果
- 比价视图: 跨平台价格对比图表
- 利润计算: 自动计算中加差价和利润率
- 一键操作: 收藏、订阅、打开商品链接、添加到比价中心

### 3.2 辅助功能

#### F6: 数据去重与匹配

- 同一商品在不同平台的自动匹配（基于 SKU / 商品名相似度）
- 价格记录去重（同天同平台同商品不重复记录）
- 数据清洗: 去除异常价格（如 $0、$999999）

#### F7: 采集成本控制

- 每日采集次数上限（可配置，初始 100 次/天）
- 按平台统计调用次数和成本
- Dashboard 展示当日/本月采集用量
- 超过阈值时提醒用户

## 4. 非功能需求

### 4.1 性能

- AI 意图解析 < 2 秒
- 单次按需采集（单平台）< 30 秒
- 跨平台比价采集（4 个平台）< 2 分钟
- 社交平台搜索 < 1 分钟
- 对话响应（非采集部分）< 1 秒

### 4.2 可靠性

- 单平台采集失败不影响其他平台结果
- 采集失败自动重试（最多 2 次）
- AI 降级: LLM 不可用时提供关键词输入 + 平台选择的 fallback UI
- 采集结果缓存: 相同查询 1 小时内不重复采集

### 4.3 反爬策略

- 请求频率控制（随机延迟 2-5 秒）
- User-Agent 轮换
- 代理 IP 池（可选配置）
- Cookie 管理（需登录平台：淘宝、1688、闲鱼、小红书、抖音等）
- Playwright stealth + 指纹伪装（Costco、Amazon 等强反爬站点）
- 商业 API 兜底（Costco Akamai 反爬时使用）
- 结果缓存减少重复请求

### 4.4 安全

- LLM API Key 加密存储
- 采集配置（API Key、Cookie）AES-256 加密存储
- 用户对话记录隔离（多用户场景）
- 采集日志脱敏

## 5. 约束条件

### 5.1 技术约束

- AI/LLM:
  - **本地大模型优先**：Ollama + Qwen2.5/Llama3/DeepSeek（零成本，隐私安全）
  - Function Calling / Tool Use（调用采集工具）
  - 可选：OpenAI / Claude 云端 API（作为高质量 fallback）
- 爬虫引擎（作为 AI 的 Tools）:
  - 加拿大电商: httpx + parsel (Walmart)、官方 API (BestBuy)、Playwright stealth (Amazon/Costco)
  - 中国电商: PriceDive (京东/淘宝/拼多多)、Alibaba-CLI-Scraper (1688)
  - 社交平台: MediaCrawler (小红书/抖音等 7 平台)、ai-goofish-monitor (闲鱼)
- 任务队列: Celery + Redis（已有 Redis）
- 数据存储: 复用现有 PostgreSQL
- 前端: React Chat 组件
- 部署: Docker 容器化

### 5.2 业务约束

- MVP 现有功能不受影响
- 爬虫模块独立部署，不影响主 API 服务
- 按需采集，控制成本（不做全量定时采集）
- 初期以公开数据采集为主

### 5.3 法律约束

- 仅采集公开可见的商品价格信息
- 遵守各平台 Terms of Service
- 不存储用户隐私数据
- LLM 调用不传输用户隐私信息

## 6. 实现优先级

> 先做 Chat + AI 核心，再逐步接入平台

| 批次         | 功能                                                                | 预计周期 | 原因                                |
| ------------ | ------------------------------------------------------------------- | -------- | ----------------------------------- |
| **Sprint 1** | F1 Chat UI + F2 AI 意图解析 + F3 BestBuy API (最简单平台验证全链路) | 4-5 天   | 先跑通 Chat→AI→采集→展示 的完整链路 |
| **Sprint 2** | F3 Amazon.ca + Walmart.ca + F5 结果展示优化（表格/卡片/图表）       | 3-4 天   | 核心加拿大平台，反爬中等            |
| **Sprint 3** | F3 京东+拼多多(PriceDive) + 1688 + 利润计算                         | 3-4 天   | 中国电商 + 利润计算场景             |
| **Sprint 4** | F3 小红书+抖音(MediaCrawler) + 闲鱼 + F4 订阅追踪                   | 4-5 天   | 社交平台 + 订阅功能                 |
| **Sprint 5** | F3 淘宝+Costco(困难平台) + F6 去重 + F7 成本控制面板 + 优化         | 4-5 天   | 反爬最严格平台 + 生产化             |
